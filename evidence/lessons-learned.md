# Lessons Learned

## Key Lessons

- Sentinel is useful only when the analyst understands what the logs mean.
- Failed logon events need context before they become a strong finding.
- KQL does not need to be advanced to be useful for junior triage.
- Grouping by account, source, and time window makes raw events easier to explain.
- Clear incident notes matter as much as the query itself.
- Screenshots should be captured during the lab, not reconstructed afterward.

## What I Would Improve Next Time

- Capture screenshots during each major lab step.
- Export sanitized query results as CSV evidence.
- Build one simple analytic rule.
- Add a workbook or map visualization if useful.
- Document setup steps while the lab is still active.

## Interview Takeaway

This lab helped me explain how IT support troubleshooting skills transfer into security operations:

- review the evidence
- ask what changed
- identify what is normal or abnormal
- document what was found
- escalate with clear notes

The biggest improvement for the next version is evidence capture discipline.
